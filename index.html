포트폴리오 웹사이트 프로젝트 설계

하드웨어 · 소프트웨어 · 시스템 시나리오 · 요구사항

1. 프로젝트 목적

개인 포트폴리오를 온라인에 공개하여 채용 담당자, 협업 파트너, 개발자 커뮤니티가 본인의 역량과 프로젝트를 쉽게 확인할 수 있도록 한다.

2. 하드웨어 구조

  • 사용자 단말: PC, 모바일, 태블릿 (웹 브라우저)
  • 서버: GitHub Pages (정적 웹 호스팅)
  • 네트워크: HTTPS 기반 요청/응답

3. 소프트웨어 구조

  • index.html : 페이지 뼈대
  • style.css : 디자인 및 반응형 UI
  • script.js : 다크 모드, 폼 처리
  • 외부 서비스 :
    • Google Fonts, Font Awesome (디자인 리소스)
    • Formspree API (연락처 폼 처리)

4. 시스템 시나리오

  1. 사용자가 브라우저에 username.github.io 입력
